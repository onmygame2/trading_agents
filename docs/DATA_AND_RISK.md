# 数据、回测与风险边界

## 数据来源

- 新浪：全市场行情快照、实时价格和按需行情。
- BaoStock：股票池及历史数据备用通道。
- AKShare：首次历史 K 线补库备用通道。
- 本地 CSV：策略、回测和 Dashboard 的主要日 K 数据源。

第三方接口没有可用性承诺，字段、限流和复权方式可能变化。任何数据源变更都应经过 `kline_sanitize.py` 校验。

## 数据门禁

系统分别检查：

- 股票池覆盖率；
- 至少 120 行的有效覆盖率；
- 相对最近交易日的新鲜覆盖率。

门禁失败时仍可生成研究报告，但不得新增组合或纸面仓位。停牌、退市和刚上市股票会导致覆盖率无法达到 100%，因此门槛按有效股票池设置。

## 回测限制

固定基准位于 `reports/backtest_v2/summary_benchmark.json`。策略改动必须比较：

- 总收益和 CAGR；
- 最大回撤；
- Sharpe；
- 胜率、交易数和盈亏比；
- 逐年及样本外表现。

当前回测不能完全消除以下偏差：

- 股票池幸存者偏差；
- 涨跌停、停牌和排队成交误差；
- 滑点与容量低估；
- 参数选择和多重检验偏差；
- 历史数据源修订。

## 推荐与成交

系统定义三个不同概念：

1. `top_picks`：研究候选；
2. `buy_picks`：经过组合权重和风控后的计划；
3. `buy_actions`：账户实际执行结果。

`preview_only` 只允许前两项，第三项必须为空。Dashboard 不应以推荐数量冒充成交数量。

## Agent 权限

Agent 可以：

- 归纳市场状态和策略表现；
- 检索历史信号与经验；
- 生成风险标记和待办建议。

Agent 不可以：

- 绕过数据门禁或账户风控；
- 直接修改成交记录；
- 使用自由文本向券商发送订单；
- 把模拟回填数据标记为真实运行结果。

## 真实资金前置条件

本项目目前仅支持纸面交易。接入真实券商前至少需要独立完成：样本外验证、长周期纸面观察、成交回写幂等性、账户级熔断、权限隔离、密钥管理、审计日志和小资金影子运行。
