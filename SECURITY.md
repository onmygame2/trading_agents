# Security Policy

## Supported version

安全修复只面向 `main` 分支的最新版本。

## 报告漏洞

请不要在公开 Issue 中披露 API 密钥、账户数据、远程代码执行、任意文件读取或未来券商适配器相关漏洞。请通过 GitHub 仓库维护者的私有联系方式报告，并提供：

- 受影响版本或 commit；
- 最小复现步骤；
- 影响范围；
- 建议修复方式（可选）。

## 密钥与隐私

- 密钥只能保存在 `.env` 或未跟踪的 `config/llm_config.json`。
- 不要提交真实账户、交易日志、SQLite 记忆库、运行日志或本地行情缓存。
- Dashboard 默认监听所有网卡时，不应直接暴露到公网；生产部署应增加认证、反向代理和网络访问控制。
- 本项目当前没有真实券商执行器。任何第三方券商扩展都必须将凭证与 Agent 上下文隔离。
